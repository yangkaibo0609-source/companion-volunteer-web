# 内容、布局与背景音乐更新设计

## 目标与基线

以远端仓库 `origin/main` 的线上故事版为实现基线，保留本地旧版快照历史。更新只覆盖用户指定的黑板数据区、志愿者留言、照片墙、游戏选择引导和背景音乐，保持既有水彩纸张、悬挂黑板、访谈票根、配色、字体和滚动节奏。

## 页面结构

所有数据黑板场景从桌面端左右双栏改为单列上下结构：上方为悬挂黑板，下方为数据纸张。黑板标签和问题文字水平、垂直居中。数据纸张继续承载正文、指标、时间线和按需加载的图表，不改变现有数据来源和指标点击交互。

数据黑板 03 保留现有正文与三个指标，之后依次插入指定图表 `c_2700ea033515c3a86b03a0f2916eb03e`、用户提供的时间来源段落和结语。它们属于同一场景的纵向阅读，不新增独立封面。跨域图表无法直接修改内部样式，因此由页面提供独立图题，并在固定高度的裁切容器中上移 iframe，隐藏图表站点内部重叠的标题区域；加载失败时保留原始图表链接。

## 内容与素材

`volunteerMessages` 的八个身份字段统一改为 `受访者01` 至 `受访者08`，回答正文不改写。照片墙新增用户提供的三张照片，转换为与现有素材一致的 WebP，加入三条循环轨道与灯箱浏览，保留原图比例并使用有意义的替代文本。

## 游戏引导

选项文字保持当前放大后的可读尺寸。选项列顶部预留引导空间，显示一个带轻微点按动画的手势符号和短提示，定位在选项区域内部而不覆盖人物、对话框、按钮或声音开关。引导只在第一幕首次出现，用户选择任一选项后隐藏；减少动态效果偏好下停止循环动画。窄屏时对话与选项改为可滚动的纵向控制区，确保最长选项完整显示。

## 背景音乐

将现有合成氛围垫音替换为用户提供的《暖光窗边》MP3，继续使用页面已有声音开关。音乐循环、低音量播放并在关闭时淡出；浏览器未获得用户手势前不自动播放。现有文字与选择提示音保留，语音卡和访谈音频行为不变。

## 组件边界

- `DataBlackboardSection` 复用现有黑板、正文和图表懒加载组件，只增加图表裁切模式和场景 03 的补充内容。
- `investigationData` 负责匿名身份与新增照片导入；`VolunteerPhotoWall` 只调整轨道索引和替代文本。
- `GameScene` 与 `ChoiceList` 增加首次选择引导状态，不改变计分、剧情和键盘选择逻辑。
- `AmbientSoundController` 只替换背景音源，保留声音开关和提示音生命周期。
- `index.css` 仅追加或调整上述组件的布局、响应式和遮挡关系。

## 异常与可访问性

图表延续 16 秒超时与外链回退。新增图片使用懒加载，灯箱继续支持 Escape 关闭和焦点回归。手势引导为装饰性元素，不加入键盘焦点；选项按钮保留数字键操作和焦点样式。背景音乐加载失败时声音开关仍可控制其他提示音，不阻断页面流程。

## 验收

1. 本地 Git 历史同时包含远端 `main` 与本地旧版快照。
2. 五个数据黑板均为上黑板、下数据，黑板文字居中，桌面和移动端无横向溢出。
3. 原图表标题不再与页面标题重叠，数据黑板 03 的新增图表、段落与结语顺序正确。
4. 八个留言身份均为匿名编号，照片墙能展示并放大新增三张照片。
5. 第一幕选择区出现手势引导且不遮挡文字，选择后隐藏；四个选项均完整可见。
6. 声音开关播放用户 MP3，关闭后停止或淡出，重新开启可继续循环。
7. TypeScript 静态检查、lint、生产构建通过；Playwright 在桌面和移动视口完成截图与遮挡检查。
